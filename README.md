Тестовое задание: каталог товаров с поиском, пагинацией, фильтрацией.
Стек: Next.js 14, TypeScript, TailWind, Axios, shadcn/ui, Lucide React (просто для красивых иконок) 

Что реализовал:
Список товаров - загружается с DummyJSON API, показывает по 10 штук, адаптивная сетка
Поиск товаров - добавил debounce, ищет в названии, описании, бренде и категории
Фильтрация по категориям - кнопки над товарами, при выборе категории сбрасывается поиск и пагинация
Страница товара - полная инфа о товаре, галерея изображений, кнопка "Назад к каталогу"
Состояния загрузки и ошибки - спиннер при загрузке, красивые страницы 404 и ошибок

Запуск: 
1. npm install
2. npm run dev
3. Открыть http://localhost:3000

Технические решения и сложности:
1. Реализовал хук useDebounce для ограничения количества запросов к api во время поиска
2. API DummyJSON использует точный текстовый поиск и он не совсем удобный (например, "phone" не находит "iPhone") -> реализовал клиентскую фильтрацию: загружаю большое количество товаров, фильтрую локально по всем полям и далее ищу подстроки (независимо от регистра)
3. Типизация категорий: изначально думал что API вернет массив строк, но вернул массив объектов, пришлось править типы и компоненты

2. Логика getProducts: функция getProducts в lib/api.ts получилась перегруженной. Она:
